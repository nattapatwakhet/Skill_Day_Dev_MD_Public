# Data Skill — โดเมนวิเคราะห์/จัดการข้อมูล

มาจาก [`work_flow_skill.md`](./work_flow_skill.md) ขั้น [3] — โดเมนระดับเดียวกับ code_skill
ใช้เมื่องานคือ **วิเคราะห์ข้อมูล / ทำความสะอาดข้อมูล / สรุปสถิติ / ทำกราฟ-ชาร์ต**

> ผสมโดเมนอื่นได้: data + code (เขียนสคริปต์วิเคราะห์) · data + deliverable (export เป็น xlsx/ชาร์ตในรายงาน) · data + research (หาข้อมูลมาวิเคราะห์)

---

## ขั้นตอน

```
1. เข้าใจข้อมูล (explore)  → มีคอลัมน์อะไร ชนิดไหน null เยอะไหม ช่วงค่าปกติ
2. ทำความสะอาด (clean)     → ซ้ำ/หาย/ค่าเพี้ยน จัดการก่อนวิเคราะห์
3. วิเคราะห์ (analyze)      → ตอบคำถามจริง ไม่ใช่สุ่มคำนวณ
4. นำเสนอ (visualize)       → เลือกชาร์ตให้ตรงกับสิ่งที่จะสื่อ
5. ตรวจ (validate)         → ตัวเลขสมเหตุผลไหม สรุปมีข้อมูลรองรับจริงไหม
```

## หลัก

- **ดูข้อมูลก่อนสรุป** — อย่าด่วนสรุปจากตัวอย่างน้อยหรือข้อมูลสกปรก
- **เลือกชาร์ตตามจุดประสงค์** — เทรนด์→เส้น, เทียบ→แท่ง, สัดส่วน→วงกลม/แท่งซ้อน, การกระจาย→histogram/scatter
- **บอกที่มา + สมมติฐาน** — กรองอะไรออก คำนวณยังไง
- **ระวัง bias** — สรุปเกินข้อมูล, ตัดข้อมูลที่ไม่เข้าทาง, correlation ≠ causation

## เครื่องมือ

- คำนวณ/วิเคราะห์ → เขียนสคริปต์ (Python: pandas/numpy) ผ่าน sandbox bash → นี่คือจุดที่ผสมกับ [`code_skill.md`](./code_skill.md)
- ทำชาร์ต → matplotlib/plotly หรือสกิล data/visualization ในระบบ ถ้ามี
- ข้อมูลจากบริการภายนอก (DB/analytics) → ค้น connector ในระบบก่อน (ดู [`permission_skill.md`](./permission_skill.md))
