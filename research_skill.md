# Research Skill — โดเมนค้นคว้า/หาข้อมูล

มาจาก [`work_flow_skill.md`](./work_flow_skill.md) ขั้น [3] — โดเมนระดับเดียวกับ code_skill
ใช้เมื่องานคือ **ค้นคว้า / หาข้อมูล / สรุปความรู้ / เช็คข้อเท็จจริง**

> ผสมโดเมนอื่นได้: research + code (หา library/วิธีแก้) · research + writing (เขียนบทความ) · research + deliverable (ทำรายงาน)

---

## ขั้นตอน

```
1. ตั้งคำถามให้คม      → จริงๆ อยากรู้อะไร ขอบเขตแค่ไหน
2. ค้นหลายแหล่ง        → อย่าเชื่อแหล่งเดียว เทียบกัน
3. สังเคราะห์ + อ้างอิง → สรุปเป็นคำตอบ ติดแหล่งที่มาทุกข้อ
4. verify             → ข้อมูลขัดกันไหม ใหม่/เก่าแค่ไหน เชื่อถือได้ไหม
```

## หลัก

- **แยกข้อเท็จจริงจากความเห็น** — ข้อมูลที่ยืนยันได้ vs การตีความ
- **ทุกข้อสรุปต้องมีแหล่ง** — ลิงก์/ที่มา ไม่ใช่ "เขาว่ากันว่า"
- **เทียบหลายแหล่ง** ก่อนสรุป โดยเฉพาะเรื่องที่เปลี่ยนตามเวลา (ราคา/เวอร์ชัน/ข่าว) → ค้นของล่าสุดเสมอ
- **บอกความมั่นใจ** — ถ้าข้อมูลไม่ชัด/ขัดแย้ง บอกตรงๆ อย่าฟันธงเกินหลักฐาน

## เครื่องมือในระบบ (เลือกตามงาน)

- ค้นเว็บทั่วไป → WebSearch / web_fetch
- ค้นเชิงลึก/หลายแหล่ง → สกิล/connector ค้นข้อมูลในระบบ (เช่น brightdata, nimble) ถ้ามี
- งานวิชาการ → สกิลงานวิจัย/ฐานข้อมูลเฉพาะทางในระบบ ถ้ามี
