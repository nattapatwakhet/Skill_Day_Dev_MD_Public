# Coding Principles — หลักคิดก่อน/ระหว่างเขียนโค้ด (hub)

มาจาก [`code_skill.md`](./code_skill.md) — ใช้คู่กับทุกงาน code (ทุกภาษา)
ไฟล์นี้เป็น **ตัวเชื่อม** ไปหลักคิด 2 สาย รายละเอียดอยู่ในไฟล์ของแต่ละสาย

| สาย | ไฟล์ | เน้น |
|---|---|---|
| คิดให้รอบคอบก่อนเขียน | [`karpathy_skill.md`](./karpathy_skill.md) | คิดก่อนเขียน, simplicity, surgical, goal-driven |
| ขี้เกียจอย่างฉลาด เขียนน้อยสุด | [`ponytail_skill.md`](./ponytail_skill.md) | บันได YAGNI, intensity, มาร์ก `ponytail:` + debt |

> สรุป 1 บรรทัด: โค้ดที่ดีที่สุดคือโค้ดที่ไม่ต้องเขียน — แต่ที่เขียนต้องถูก, แตะเท่าที่ขอ, และมีของตรวจ

ทั้งสองเสริมกัน: karpathy = คิด/ขอบเขต/เป้าวัดผล, ponytail = ตัดให้เหลือน้อยสุด
รีวิวหา over-engineering ของโค้ดที่มีอยู่ → [`review_skill.md`](./review_skill.md)
