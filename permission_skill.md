# Permission Skill

มาจาก [`work_flow_skill.md`](./work_flow_skill.md) ขั้น [3.3] — เปิดไฟล์นี้ก่อนขอสิทธิ์/แตะเครื่อง user ทุกครั้ง

เรื่อง "ต้องขอสิทธิ์อะไรบ้าง" และ "อะไรทำได้/ทำไม่ได้" เวลาทำงานใน workspace ของ user
อ่านไฟล์นี้ก่อนขอสิทธิ์ทุกครั้ง

> ส่วน "วิธีทำงาน/ลำดับการคุมจอ/รันคำสั่ง" อยู่ในไฟล์ [`controlled_operation_skill.md`](./controlled_operation_skill.md) แยกต่างหาก

---

## สิทธิ์มี 3 แบบ แยกกัน

### 1. สิทธิ์เข้าถึงโฟลเดอร์ (อ่าน/เขียน/ลบไฟล์)

- user เลือกโฟลเดอร์หรือ workspace ที่แชร์ให้
- อ่าน/แก้ไฟล์ได้ **เฉพาะในโฟลเดอร์ที่แชร์เท่านั้น** นอกนั้นเข้าไม่ถึง (เช่น `uploads` = read-only)
- **ลบไฟล์**: ต้องขอ user ก่อนทุกครั้ง แม้ไฟล์อยู่ใน workspace ที่แก้ได้
- ต้องการโฟลเดอร์เพิ่ม ให้ขอสิทธิ์จาก user ก่อน ไม่เดา path เอง

### 2. สิทธิ์คุมหน้าจอ (computer use) — ขอเป็น "รายแอป"

ถ้าต้องคุมหน้าจอหรือแอป ให้ขอสิทธิ์ตามเครื่องมือที่ runtime รองรับ แล้วรอ user อนุญาตก่อน
แต่ละ runtime อาจมีระดับสิทธิ์ไม่เท่ากัน ห้ามหาทางลัดเลี่ยงหรือใช้วิธีนอกระบบเพื่อยกระดับสิทธิ์:

| แอป | ระดับ | ทำอะไรได้ |
|---|---|---|
| Finder | **full** | คลิก ดับเบิลคลิก พิมพ์ ดู คุม ค้นหา — ได้หมด |
| Terminal / IDE (VS Code, iTerm ฯลฯ) | **click** | เห็น + คลิกซ้ายได้เท่านั้น — **พิมพ์ไม่ได้** paste ไม่ได้ คลิกขวาไม่ได้ |
| Browser (Edge, Chrome, Safari) | **read** | เห็นบนจอได้อย่างเดียว — คลิก/พิมพ์/เปิด URL ไม่ได้ |

> ถ้าต้องคุมเบราว์เซอร์จริง ให้ใช้ browser automation/tool ที่ runtime นี้รองรับ — ดู `controlled_operation_skill.md`

### 3. สิทธิ์เชื่อมบริการภายนอก (connectors / MCP)

- Gmail, Calendar, Slack, Drive ฯลฯ ต้องเชื่อมต่อแยกอีกที
- ถ้างานต้องใช้บริการภายนอก ให้ใช้ connector/tool ที่มีใน runtime หรือเสนอให้ user เชื่อมต่อก่อน

## สิ่งที่ "เปิดเต็มแค่ไหนก็ทำไม่ได้" (กฎความปลอดภัย ห้ามเด็ดขาด)

- กรอกรหัสผ่าน / เลขบัตร / ข้อมูลการเงิน / SSN / API key
- โอนเงิน / เทรด / ซื้อ-ขายสินทรัพย์
- ลบถาวรระดับระบบ (empty trash, hard-delete อีเมล/ข้อความ) — *ยกเว้น* ลบไฟล์ในโฟลเดอร์ที่แชร์ซึ่งเปิดสิทธิ์แล้ว ทำได้แต่ต้องขอก่อนทุกครั้ง
- เปลี่ยนการตั้งค่าความปลอดภัยหรือสิทธิ์การแชร์
- ทำ CAPTCHA / bot-detection
- ให้คำแนะนำการลงทุนแบบเจาะจง

→ พวกนี้ให้ **บอก user ทำเอง** ไม่ทำแทน

---

## สิ่งที่ "ต้องขออนุญาตทุกครั้งก่อนทำ"

- ส่งข้อความ/อีเมล/คอมเมนต์แทน user
- โพสต์/เผยแพร่/แก้เนื้อหาสาธารณะ
- ซื้อของด้วยวิธีจ่ายที่บันทึกไว้
- ดาวน์โหลดไฟล์ (บอกชื่อไฟล์/แหล่ง/ขนาด)
- กดปุ่มที่ย้อนกลับไม่ได้ (ส่ง / ยืนยัน / ลบ / publish)
- กรอก/ส่งฟอร์มที่มีข้อมูลส่วนตัว
- ดู/คุมหน้าจอ (ขอ user ก่อนทุกครั้ง — ดูกฎใน `controlled_operation_skill.md`)
