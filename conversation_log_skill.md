# Conversation Log Skill — บันทึกสิ่งที่คุยในแชต

มาจาก [`skill_maintenance_skill.md`](./skill_maintenance_skill.md) — ใช้เมื่อ user ขอให้
**สรุป/อัปเดตสิ่งที่คุยในแชตลง skill** หรือถามว่า "คุยอะไรไปบ้าง / อะไรยังไม่ได้ลง skill"

ไฟล์นี้เป็น log ของ session และรายการ follow-up สำหรับชุด skill เท่านั้น ไม่ใช่ที่เก็บกฎถาวรทั้งหมด
ถ้าข้อไหนกลายเป็นกฎที่ใช้ซ้ำ ให้ย้ายไปไฟล์ skill เฉพาะโดเมน แล้วเหลือแค่ลิงก์อ้างอิงไว้ที่นี่

> ต่างจาก [`progress_skill.md`](./progress_skill.md) (สถานะงานที่กำลังทำ) — ไฟล์นี้ = log ประวัติการคุย +
> สิ่งที่ sync เข้า skill แล้ว/ยังไม่ได้ลง

> ไฟล์นี้เริ่มมาแบบ **log เปล่า** — คนที่นำชุดนี้ไปใช้เริ่มจดของตัวเองได้เลย (เนื้อหาตัวอย่างด้านล่างลบทิ้งได้)

---

## วิธีใช้เมื่อ user ขอ sync chat → skill

1. อ่าน [`main_skill.md`](./main_skill.md) และ [`work_flow_skill.md`](./work_flow_skill.md) ก่อนเสมอ
2. อ่าน [`skill_maintenance_skill.md`](./skill_maintenance_skill.md) เพื่อเช็กกฎแก้ไฟล์ skill
3. แยกของที่คุยในแชตเป็น 3 กลุ่ม:
   - **ลง skill แล้ว** — มีไฟล์รองรับและเนื้อหาครอบคลุมแล้ว
   - **ยังไม่ได้ลง** — เป็นกฎ/โครงสร้าง/แพตเทิร์นที่ควรจำ แต่ยังไม่มีในไฟล์ที่เหมาะ
   - **ไม่ควรลง skill** — คำสั่งครั้งเดียว, ข้อมูลส่วนตัว, secret, หรือบริบทชั่วคราวที่ไม่ควรเก็บ
4. ถ้าเป็นกฎถาวร ให้เพิ่มในไฟล์โดเมนที่ถูกต้อง ไม่เก็บซ้ำใน log
5. ถ้าเป็นสรุปแชตหรือรายการค้าง ให้เพิ่มใน "บันทึกแชต" ด้านล่าง พร้อมวันที่
6. ถ้าเพิ่มไฟล์ skill ใหม่ ต้องอัปเดต index ใน [`main_skill.md`](./main_skill.md) และจุดที่เกี่ยวใน workflow

## บันทึกแชต

> เริ่มจดจากตรงนี้ — แต่ละครั้งที่ user ขอ sync ให้เพิ่ม entry ใหม่ (ล่าสุดไว้ล่างสุด) พร้อมวันที่
> ตัวอย่างฟอร์แมตด้านล่างเป็น template ลบ/แทนที่ด้วยของจริงได้เลย

### YYYY-MM-DD — <หัวข้อสั้นๆ ของสิ่งที่คุย>

- **คุยอะไรไป:** <สรุปสั้นๆ ว่า session นี้ทำ/ถกอะไร>
- **ลง skill แล้ว:** <ไฟล์ + เรื่องที่มีอยู่แล้ว ไม่ต้องเพิ่มซ้ำ>
- **ยังไม่ได้ลง → เพิ่มรอบนี้:** <กฎ/แพตเทิร์นใหม่ + ไฟล์โดเมนที่เพิ่มเข้าไป>
- **ไม่ควรลง skill:** <คำสั่งครั้งเดียว / ข้อมูลส่วนตัว / secret / payload จริง>
