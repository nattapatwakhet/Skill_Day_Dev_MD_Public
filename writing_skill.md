# Writing Skill — โดเมนเขียนเนื้อหา

มาจาก [`work_flow_skill.md`](./work_flow_skill.md) ขั้น [3] — โดเมนระดับเดียวกับ code_skill
ใช้เมื่องานคือ **เขียนเนื้อหา** — บทความ, โพสต์, อีเมล, README, doc, changelog, copy

> ผสมโดเมนอื่นได้: writing + code (doc/changelog/คอมเมนต์) · writing + research (บทความมีอ้างอิง) · writing + deliverable (จัดรูปเป็น docx/สไลด์)

---

## ขั้นตอน

```
1. ใคร่อ่าน + เป้าหมาย   → เขียนให้ใครอ่าน อยากให้เขาทำ/รู้อะไร
2. วางโครง               → ลำดับความคิด หัวข้อหลัก
3. ร่าง                  → เขียนให้ครบก่อน ยังไม่ต้องเป๊ะ
4. ขัด (tighten)         → ตัดคำเกิน ประโยคยาว→สั้น คำคลุมเครือ→ชัด
```

## หลัก

- **รู้ผู้อ่านก่อนเขียน** — dev / ลูกค้า / ผู้บริหาร โทนและศัพท์ต่างกัน
- **นำสาระก่อน** — ใจความสำคัญขึ้นต้น อย่าให้คนอ่านขุด
- **กระชับ** — ตัดได้แล้วได้ใจความเท่าเดิม = ตัด (ทดสอบแบบเดียวกับ ponytail แต่กับคำ)
- **active voice, ประโยคสั้น, รูปธรรม** — "กดบันทึกแล้วระบบเซฟทันที" ดีกว่าอ้อม
- **ความสม่ำเสมอ** — ศัพท์/โทนเดียวกันทั้งชิ้น

## โยงกับสกิลอื่น

- เขียนให้ **คนไม่ใช่ dev / ผู้บริหาร** (รายงานสถานะ ตามช่องทาง) → [`report_skill.md`](./report_skill.md)
- เขียน **copy ใน UI** (ปุ่ม/error/empty state) → [`frontend_design_skill.md`](./frontend_design_skill.md)
- เขียน **เอกสาร/สเปกแบบ co-author** หรือจัดรูปเป็นไฟล์ → [`deliverable_skill.md`](./deliverable_skill.md)
- บทความที่ต้องมีข้อมูล/อ้างอิง → คู่กับ [`research_skill.md`](./research_skill.md)

> ตรวจงานเขียนก่อนส่ง (เนื้อหาตรงเป้า ข้ออ้างมีหลักฐาน) → [`review_skill.md`](./review_skill.md)
